# GYEOL 서비스형 프로토타입 흐름

## 제품 원칙

결의 본체는 지도 접힘 영상이 아니라 다음 질문에 근거와 함께 답하는 위치 해독 서비스입니다.

> 제가 지금 있는 곳은 대동여지도에서 어디였나요?

지도 연출은 답변을 확인한 사용자가 선택하는 후속 체험으로만 사용합니다. 천안은 공식 판면 확대, 서울은 기존 접이 지도와 김정호 보행으로 분리합니다.

## 서비스 상태

### 1. 현재 위치

- 장소 검색
- 현재 위치 권한 요청
- 검증된 시범 장소 선택
- 미지원 위치는 결과를 생성하지 않고 지원 범위를 안내

### 2. 옛지명 해독

1. 현대 장소와 행정구역 확인
2. 옛지명 권역 후보 대조
3. 실제 판본 정보 확인
4. 판본·공공데이터·추정을 분리한 답변 구성

정적 데모에서 근거 없는 분석 퍼센트나 AI 신뢰도 숫자는 표시하지 않습니다.

### 3. 근거 확인

결과는 답변 문장 하나가 아니라 다음 구조를 유지합니다.

- 현대 장소
- 옛지명 권역 후보
- 후보가 나온 이유
- 판본 근거
- 공공데이터 근거
- 공간·명칭 대응 추정
- 정확한 한 점이 아니라는 한계

### 4. 지도 체험

- 천안역 결과: `천안군 권역 판본 확대하기`를 선택하면 전체 지도에서 국립중앙박물관 공식 이미지 파일 96의 천안 표기로 전환합니다. 현대 천안역과 옛 지도의 천안 표기를 같은 점으로 표시하지 않습니다.
- 서울 결과: `한성부 권역 지도 체험하기`를 선택한 뒤에만 기존 GSAP 지도 전개 화면을 불러옵니다. 먹길은 검증된 역사 경로가 아니므로 `한성부 권역 시각화 체험`으로만 표현합니다.

## 현재 시범 범위

- 천안역
- 서울시청
- 광화문
- 경복궁
- 결과: 천안군 권역 후보 또는 한양·한성부 권역 후보

시범 결과는 실제 서비스 구조와 상호작용을 검증하기 위한 데이터입니다. 국립중앙박물관 소장품 `신수19997`, 공공데이터포털 GW 데이터셋 `15159017`, 한국철도공사 역 위치 데이터셋 `15127532`, 국가철도공단 철도역 정보 데이터셋 `15067652`, 천안시 공식 연혁을 출처로 등록했습니다. 천안 전체지도 점과 공식 이미지 파일 96의 표기점 모두 시각적 근사값이며 지오리퍼런싱 결과나 정확한 역사 좌표가 아닙니다.

## 근거 전달 상태

- `공식 원문 검수본`: 2026-07-11에 국립중앙박물관 소장품 원문과 이용조건을 확인한 앱 내 저장본
- `API 갱신 저장본`: 서버 또는 CI에서 GW API를 호출하고 허용 필드만 정규화한 저장본
- `API 연결 준비`: 데이터셋과 명세는 연결했지만 서버 인증키로 아직 갱신하지 않은 상태

브라우저에서는 공공데이터 인증키를 직접 사용하지 않습니다. 정적 배포에서는 `npm run sync:emuseum`으로 서버/CI에서 저장본을 먼저 갱신한 뒤 빌드합니다.

## 다음 데이터 작업

공식 도성도와 천안 공식 이미지 파일 96 원본, 출처 계보, 서울·천안 추정점을 각 체험에 연결했습니다. 다음 검수 단계는 아래와 같습니다.

1. 역사 지도 전문가가 도성도 3개 기준점을 검토하고 `approximate` 유지 또는 승격 결정
2. 서울시청 추정점에 사용한 군기시 유구·도성 남북축 근거 레코드 보강
3. 역사 지도 전문가가 공식 이미지 파일 96의 천안 표기점과 크롭 영역을 검토하고 지오리퍼런싱 여부 결정
4. 공식 전체 대동여지도 첩·면과 한성부·천안 권역점을 연결하는 별도 좌표 등록
5. 배포 환경이 서버를 제공하면 저장본 방식에서 BFF 방식으로 확장
6. 서비스가 구성한 시각화 경로와 실제 역사 역로를 분리해 검증

상세한 식별자, API 갱신 방법, 키 보안 규칙은 `docs/evidence-data.md`에 기록합니다.
