# 서비스 설계안

결(GYEOL)은 발표용 인터랙티브 데모에서 끝나는 프로젝트가 아니라, 실제 사용자가 자신의 위치와 질문을 통해 대동여지도를 읽을 수 있게 만드는 문화유산 해독 서비스로 설계합니다.

## 서비스 포지션

한 줄 정의:

> 현재 위치와 질문을 1861년 대동여지도의 옛지명, 길, 기호, 판본 근거로 번역해주는 근거형 AI 문화유산 서비스

서비스의 핵심 가치는 다음입니다.

- 사용자는 고지도를 몰라도 질문할 수 있다.
- 서비스는 답변 전에 실제 판본과 공공 문화데이터 근거를 찾는다.
- 불확실한 내용은 단정하지 않고 후보와 신뢰도로 보여준다.
- 시각적으로는 몰입감 있게, 정보 구조는 검증 가능하게 유지한다.

## 핵심 사용자

| 사용자 | 주요 상황 | 핵심 가치 |
| --- | --- | --- |
| 일반 관람객 | 박물관, 전시, 웹에서 대동여지도 탐색 | 어려운 고지도를 쉽게 이해 |
| 지역주민 | 내 동네가 옛 지도에서 어떻게 보였는지 궁금함 | 현재 위치와 역사 연결 |
| 학생/교사 | 수업, 체험학습, 발표자료 제작 | 질문 기반 학습 |
| 외국인 관람객 | 한국 고지도 이해가 어려움 | 다국어 음성 도슨트 |
| 박물관/지자체 | 전시·관광 콘텐츠 필요 | 데이터 기반 해설 서비스 |

## 1차 서비스 범위

처음부터 전국 전체를 자동화하지 않습니다. 서비스 MVP는 "잘 되는 한 장면"을 먼저 제품화합니다.

1차 범위:

- 지역: 천안, 서울, 부산
- 지도: 대동여지도 1첩 또는 발표 핵심 권역
- 질문: 대표 질문 20개
- 기호: 도로, 읍치, 역참, 산줄기, 하천 등 5개
- 입력: 위치 검색, 현재 위치, 직접 질문
- 출력: 옛지명 후보 3개, 판본 영역, 근거 카드, 신뢰도 라벨, 자연어 답변

## 핵심 인터랙션

서비스의 첫 화면은 전체 대동여지도입니다. 사용자는 지명을 검색하거나 지도 위 권역을 직접 클릭해 관심 지역으로 들어갑니다.

```text
전체 대동여지도
  ↓
검색 또는 지도 클릭
  ↓
관련 권역 확대
  ↓
옛지명 후보 표시
  ↓
판본 근거 / 공공데이터 근거 / 추정 라벨 확인
  ↓
AI 도슨트 답변
```

이 구조는 박물관 제안에 적합합니다. 관람객은 먼저 대동여지도 전체의 물성을 보고, 자신의 지역을 선택한 뒤, 확대된 권역에서 옛지명과 근거를 확인합니다.

## 대표 사용자 흐름

### 현재 위치 기반

1. 사용자가 "내 위치로 보기"를 누른다.
2. 서비스가 현대 좌표와 행정구역을 확인한다.
3. 주변 POI와 옛지명 후보를 찾는다.
4. 대동여지도 판본의 관련 영역을 보여준다.
5. 후보 3개와 신뢰도를 표시한다.
6. 판본 근거, 공공데이터 근거, 추정 라벨을 분리한다.
7. AI 도슨트가 단정하지 않는 문장으로 답한다.

### 검색 기반

1. 사용자가 "천안역"을 검색한다.
2. 서비스가 좌표와 현대 행정구역을 찾는다.
3. 옛 행정권역 후보를 계산한다.
4. 지도 위에 현대 위치와 옛지도 후보 영역을 연결한다.
5. 사용자는 후보별 근거와 신뢰도를 비교한다.

### 질문 기반

예시 질문:

- 제가 지금 있는 곳은 대동여지도에서 어디였나요?
- 여기서 한양까지는 어떤 길을 따라갔나요?
- 이 기호는 무슨 뜻인가요?
- 이 지역 주변에는 어떤 읍치나 역참이 있었나요?

질문형 답변은 반드시 검색된 근거 묶음을 바탕으로 생성합니다.

## 제품 화면 구조

### 1. 몰입형 지도 홈

첫 화면은 설명 페이지가 아니라 바로 사용할 수 있는 지도 경험이어야 합니다.

- 전체 화면에 실제 대동여지도 판본 이미지
- 상단 검색창: 장소 또는 질문 입력
- 현재 위치 버튼
- 좌측 현대 지도 미니 패널
- 우측 근거형 답변 패널
- 하단 후보 타임라인 또는 신뢰도 바

시각 톤:

- 어두운 전시 공간 같은 배경
- 실제 판본은 따뜻한 종이 질감으로 크게 노출
- 후보선, 신뢰도, 데이터 레이어는 빛나는 얇은 선으로 표현
- 카드 UI는 화려하되 출처와 추정 라벨이 눈에 띄어야 함

### 2. 후보 비교 패널

후보 3개를 카드로 보여줍니다.

카드 구성:

- 옛지명
- 현재 행정구역 연결
- 신뢰도 점수
- 근거 개수
- "권역 단위 추정" 라벨
- 판본 영역 보기 버튼

### 3. 근거형 도슨트 패널

답변 패널은 항상 근거와 함께 나옵니다.

구성:

- 답변 요약
- 판본 근거
- 공공데이터 근거
- 추정/주의 라벨
- 더 보기: 원문 출처, 데이터 기준일, 신뢰도 계산 설명

## 기술 구조

```text
Web / Mobile UI
  ↓
Service API
  ↓
GYEOL Engine
  ├─ Geo Resolver
  ├─ Historical Place Matcher
  ├─ Evidence Retriever
  ├─ Confidence Scorer
  └─ Answer Composer

  ├─ Modern Places

  ├─ Map Regions
  ├─ Evidence Sources
  └─ Question Templates
```

## 신뢰도 정책

신뢰도는 AI가 임의로 정하지 않습니다. 규칙 기반 점수로 시작합니다.

예시:

```text
score =
  distance_score * 0.40
+ admin_region_score * 0.25
+ name_similarity_score * 0.20
+ evidence_score * 0.15
```

라벨:

- 0.75 이상: 가능성이 높습니다
- 0.55 이상 0.75 미만: 후보로 볼 수 있습니다
- 0.55 미만: 단정하기 어렵습니다

금지:

- 정확한 한 점이라고 말하기
- 근거 없는 역사 단정
- AI가 지도를 복원했다고 표현
- 출처 없는 수치 제시

## AI 사용 원칙

AI는 판단자가 아니라 설명자입니다.

코드와 데이터가 하는 일:

- 장소 후보 계산
- 근거 검색
- 신뢰도 산출
- 답변에 넣을 사실 묶음 구성

AI가 하는 일:

- 사용자 질문 의도 분류
- 근거 묶음을 쉬운 문장으로 설명
- 어린이/외국인/고령층용 말투 변환
- 긴 근거를 짧게 요약

AI 입력에는 항상 구조화된 근거만 전달합니다.

```json
{
  "query": "천안역",
  "candidate": "천안군",
  "confidence": 0.78,
  "label": "권역 단위 추정",
  "evidence": [
    "국립중앙박물관 대동여지도 신수19997",
    "국가유산청 고지도·지도표 해설",
    "문화공공데이터"
  ],
  "answer_policy": "정확한 한 점으로 단정하지 말 것"
}
```

## 수익화/운영 모델

서비스는 B2C와 B2B2C를 함께 고려합니다.

가능한 모델:

- 무료 웹 서비스: 공공성과 홍보 중심
- 박물관/지자체 전시용 라이선스
- 교육기관용 수업 모드
- 지역 관광 콘텐츠 패키지
- 다국어 AI 도슨트 API

초기에는 공모전/PoC 성격에 맞춰 무료 웹 데모와 기관 제안용 패키지를 우선합니다.

## 구현 우선순위

1. 천안역 핵심 흐름을 실제 화면으로 완성
2. 실제 판본 이미지 뷰어와 후보 영역 표시
3. 후보 3개와 신뢰도 카드
4. 근거 카드와 답변 패널
5. 검색 입력과 현재 위치 입력
6. 대표 질문 20개
7. 음성 입력/STT
8. 다국어/TTS
9. 관리자용 데이터 검수 화면

## 서비스 성공 기준

MVP가 성공하려면 사용자가 다음을 1분 안에 이해해야 합니다.

- 내가 입력한 현대 위치가 옛지명 후보로 바뀐다.
- 후보는 실제 대동여지도 판본 영역과 연결된다.
- 답변에는 근거와 신뢰도가 있다.
- 불확실한 내용은 추정이라고 표시한다.
