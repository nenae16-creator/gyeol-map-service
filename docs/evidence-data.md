## 한성 관련 공식 이미지

- 갤러리 7번 `도성도`: `ssu019997-00-08.jpg`, 3000×1825
- 갤러리 8번 `경조오부도`: `ssu019997-00-09.jpg`, 3000×1815

국립중앙박물관 공식 설명에서 7번 화면을 `대동여지도(신수19997) 중 도성도`로 식별합니다. 현재 서비스의 `daedongyeojido-idle-toned-v2.png`는 이 공식 원본이 아니라 프로토타입용 파생 시각 자산입니다.

지도 체험의 확대 화면은 공식 공개 이미지 URL의 원본 픽셀을 그대로 보존한 다음 자산을 사용합니다.

```text
public/assets/official/nmk-shinsu19997-doseongdo-original.jpg
3000×1825 · 5,573,982 bytes
SHA-256 F59EDC35FF78E57A7A4EB08BC6790597308C4E8F1664A0DF43F324776BED802A
```

취득 경로·원문·권리·해시는 `public/assets/official/nmk-shinsu19997-doseongdo.provenance.json`에 기록했습니다. 픽셀을 변형한 파생 파일은 만들지 않았고, 크롭·그림자·표식·경로는 CSS/SVG 레이어로만 적용합니다. 기관 납품용 원본 보관 절차가 필요하면 소장품 페이지의 사용목적 기록형 내려받기 UI로 다시 확보하고 같은 해시인지 확인합니다.

## 천안 관련 공식 이미지와 현대 근거

천안 체험은 국립중앙박물관 신수19997의 공식 이미지 파일 96 원본을 사용합니다. 파일명의 `96`은 공식 면·엽 번호로 단정하지 않고 공개 이미지 식별 문맥으로만 표시합니다.

```text
public/assets/official/nmk-shinsu19997-cheonan-sheet-96-original.jpg
원본 파일명 ssu019997-00-96.jpg
3000×1794 · 5,744,880 bytes
SHA-256 CCCC9342B409E5BC242D04CD275370C7EFEE2AAB56BE56324871895ABE9B087D
```

원본 판면에서 `천안(天安)` 표기를 시각 판독했습니다. 취득 경로·공공누리 제1유형·원본 해시는 `public/assets/official/nmk-shinsu19997-cheonan-sheet-96.provenance.json`에 기록합니다. 화면 크롭과 표식은 CSS 레이어이며 원본 픽셀은 수정하지 않습니다.

현대 위치와 역사 명칭은 다음 공식 자료를 함께 사용합니다.

| 구분 | 근거 |
| --- | --- |
| 현대 역 대표점 | [한국철도공사 역 위치 정보](https://www.data.go.kr/data/15127532/fileData.do), 데이터셋 `15127532`, 천안역 `(36.808837, 127.1471716)` |
| 현대 주소·개업 | [국가철도공단 철도역 정보](https://www.data.go.kr/data/15067652/fileData.do), 데이터셋 `15067652`, `충청남도 천안시 동남구 대흥로 239`, `1905-01-01` 개업 |

| 판본 표기 | [국립중앙박물관 대동여지도 신수19997](https://www.museum.go.kr/site/main/relic/search/view?relicId=4502) 공식 이미지 파일 96의 `天安` |

현대 천안역은 1861년 판본 제작 이후 생긴 시설입니다. 따라서 서비스는 역 대표점과 판본의 천안 표기를 같은 점으로 연결하지 않고 `천안군 권역 후보`로만 제시합니다.

## 공공데이터 GW API

- 데이터셋: `문화체육관광부 국립중앙박물관_전국 박물관 유물정보_GW`
- 데이터셋 ID: `15159017`
- Base URL: `https://apis.data.go.kr/1371027/openapi`
- 엔드포인트: `/code`, `/list`, `/detail`
- 응답: XML
- 인증: 공공데이터포털 `serviceKey`
- 공식 명세: <https://www.data.go.kr/data/15159017/openapi.do>

동기화 스크립트는 목록에서 다음 조합을 모두 확인한 뒤 상세 레코드를 가져옵니다.

```text
id = PS0100100101101999700000
relicNo = 19997
museumName3 = 신수
```

외부 응답 전체를 저장하지 않습니다. 이름, 작가, 소장품번호, 소장기관, 시대, 재질, 크기, 설명, 이미지 수, 권리 코드만 새 객체로 복사합니다.

## 저장본 갱신

PowerShell 예시:

```powershell
$env:DATA_GO_KR_SERVICE_KEY="발급받은_키"
npm run sync:emuseum
npm run verify:evidence
npm run build
```

키는 `.env`나 배포 환경의 서버 비밀값으로만 관리합니다. `VITE_` 접두사를 붙이면 클라이언트 번들에 포함될 수 있으므로 사용하지 않습니다. 스크립트는 요청 URL, 응답의 `params`, 인증키를 출력하거나 저장하지 않습니다.

갱신 대상은 `src/data/snapshots/nmk-shinsu19997.v1.json`입니다. 키가 없거나 API 조회·검증에 실패하면 기존 저장본을 변경하지 않습니다.

## 좌표 상태

결과 화면의 전체지도 마커는 기존 시안 좌표를 복원한 값입니다.

현재 한성부 마커는 로컬 512×767 자산의 다음 점을 사용합니다.

```text
normalized = (0.349695, 0.440690)
pixel = (179.04, 338.01)
status = approximate
```

이는 기존 시안 표식을 복원한 값이며 도성도 원본의 지오리퍼런싱 결과, 서울시청의 역사 좌표, 한성부 경계가 아닙니다. 공식 도성도 원본을 확보하고 사람 검수를 마칠 때만 `human-verified`로 승격합니다.

지도 체험의 도성도에는 원본 3000×1825 좌표 공간을 기준으로 다음 점을 별도 등록했습니다.

| 장소 | 원본 정규화 좌표 | 상태 | 의미 |
| --- | --- | --- | --- |
| 경복궁 | `(0.380667, 0.318904)` | `approximate` | 도상 시각 판독점 |
| 광화문 | `(0.382333, 0.375890)` | `approximate` | 경복궁 남문 도상 시각 판독점 |
| 서울시청 | `(0.388667, 0.649315)` | `approximate` | 군기시 터 인근에 투영한 현대 위치 추정점 |

경복궁·광화문도 아직 지오리퍼런싱이나 외부 전문가 검수를 마치지 않았으므로 `human-verified`로 표시하지 않습니다. 서울시청은 1861년 지도에 존재하지 않는 현대 시설이므로 정확한 역사 좌표라고 표현하지 않습니다.

서울시청 추정의 보조 근거는 서울특별시 문화포털 `군기시유적전시실` 안내입니다. 이 자료는 서울시청 건립공사 중 조선시대 군기시 유구가 출토됐음을 확인하는 출처일 뿐, 도성도 위 한 점의 정확성을 보증하는 지오리퍼런싱 자료가 아닙니다.

천안은 두 개의 `approximate` 영역을 분리해 등록합니다.

| 영역 | 정규화 좌표 | bounds | 의미 |
| --- | --- | --- | --- |
| 로컬 전체지도 | `(0.369, 0.507)` | `(0.319, 0.467, 0.10, 0.08)` | 전체 형상에서 천안 일대를 찾기 위한 시각적 근사점 |
| 공식 이미지 파일 96 | `(0.457333, 0.604236)` | `(0.32, 0.36, 0.30, 0.48)` | 원본 판면의 `天安` 표기를 시각 판독한 점과 크롭 |

두 값 모두 픽셀 좌표이며 경위도나 역사 행정경계가 아닙니다. 전문가 검토와 지오리퍼런싱 전에는 `human-verified`로 승격하지 않습니다.

먹길은 공식 지도 이미지에 합성 저장하지 않습니다. 서비스가 구성한 SVG 경로로 별도 렌더링하며 김정호의 실제 답사 경로를 주장하지 않습니다.

## 현대 직선거리 기준

현대 거리의 출발 기준은 `공주시청`으로 고정합니다.

| 항목 | 값 |
| --- | --- |
| 기준지 | 공주시청 |
| 공식 주소 | 충청남도 공주시 봉황로 1 |
| WGS84 좌표 | `(36.446542105093314, 127.11922678155355)` |
| 공식 주소 출처 | <https://www.gongju.go.kr/kr/sub05_04_01.do> |
| 좌표 출처 | 공식 페이지의 `data-lat`, `data-lng` 내장 지도 값 |
| 접근일 | 2026-07-13 |

서울 접이 지도 체험의 목적지는 화면에서 실제로 도착하는 `DemoPlace`의 현대 위·경도를 사용합니다. 브라우저 현재 위치는 시범 지도에서 서울시청 권역점으로 대리 표시하므로 현대 직선거리도 서울시청 기준점까지 계산합니다. 사용자 좌표까지의 거리로 오해하지 않도록 `현재 위치` 좌표를 거리값에 직접 사용하지 않습니다. 천안 판면 확대 체험은 현대 거리나 `/api/road-distance`를 요청하지 않습니다.

거리는 평균 지구 반지름 `6,371,008.8m`를 사용한 Haversine 대권거리로 계산합니다. 화면에는 반드시 `현대 좌표 직선거리`, `약`, `도로 이동거리와 다름`을 함께 표시합니다. 이 값은 도로·도보 거리, 1861년 노정 거리, SVG 먹길 길이, 체험 30칸과 서로 환산하지 않습니다.

## 현대 자동차 추천 경로

자동차 도로거리와 예상시간은 [카카오모빌리티 자동차 길찾기](https://developers.kakaomobility.com/guide/navi-api/directions)의 `RECOMMEND` 경로 요약값을 사용합니다.

| 항목 | 값 |
| --- | --- |
| 서버 엔드포인트 | `/api/road-distance?destinationId=...` |
| 외부 호출 | `GET https://apis-navi.kakaomobility.com/v1/directions` |
| 인증 | 서버 전용 `KAKAO_MOBILITY_REST_API_KEY` |
| 경로 기준 | `priority=RECOMMEND`, `summary=true` |
| 거리 | `routes[0].summary.distance` 미터 |
| 예상시간 | `routes[0].summary.duration` 초 |
| 서버 캐시 | 2분, 이후 3분 동안 stale 재검증 허용 |

브라우저는 목적지 ID만 동일 출처 서버 함수로 전달합니다. 서버는 공주시청과 시범 목적지 3곳을 허용 목록으로 관리하며 임의 좌표 프록시로 동작하지 않습니다. REST API 키와 카카오 인증 헤더는 클라이언트 번들·응답·진단 정보에 포함하지 않습니다.

자동차 추천 경로는 교통상황과 제공자의 경로 정책에 따라 달라질 수 있으며 도보거리나 1861년 노정 거리가 아닙니다. 서버 키가 설정되지 않았거나 제공자 요청이 실패해도 직선거리·고지도 상태·지도 체험은 유지하고 자동차 경로만 `현재 확인할 수 없음`으로 표시합니다.

## 이용조건 분리

- 소장품 페이지와 판본 이미지: 공공누리 제1유형, 출처표시
- GW API 데이터셋 메타데이터: 공공누리 제0유형, 자유이용
- 한국철도공사 역 위치 데이터: 이용허락범위 제한 없음
- 국가철도공단 철도역 정보: 이용허락범위 제한 없음
- 천안시 연혁: 공식 사실 확인 출처로 링크와 접근일 표시

API 데이터셋의 제0유형을 판본 이미지에 적용하지 않습니다. 화면에는 판본 이미지 기준으로 보수적인 제1유형 출처표시를 유지합니다.
