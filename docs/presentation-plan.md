# 발표 구성

발표 시간은 10분, PPT는 12~14장 구성을 기준으로 합니다. 핵심은 예쁜 영상보다 한 장면의 이해입니다.

## 필수 화면

### 1. 한 문장 정의

화면 문구:

> 결(GYEOL)  
> 현재 내 위치를  
> 1861년 대동여지도의 기호·길·지명으로 번역하는  
> 공공 문화데이터 해독 엔진

시각 구성:

- 왼쪽: 현대 지도 또는 천안역 이미지
- 오른쪽: 대동여지도 이미지
- 가운데: 번역 화살표

### 2. 문제 장면

핵심 문장:

> 대동여지도는 보이지만 읽히지 않는다.

시각 구성:

| 기존 전시 / 고해상 지도 | 관람객 질문 |
| --- | --- |
| 크게 볼 수 있음 | 그런데 이 기호가 뭔지 모름 |

넣을 문장:

> 고해상으로 보는 경험은 이미 있다. 하지만 관람객은 아직 지도를 읽지 못한다.

### 3. 핵심 데모 장면

이 장이 발표에서 가장 중요합니다.

흐름:

1. 현재 위치: 천안역
2. 현대 행정구역·POI
3. 옛지명 후보 3개
4. 대동여지도 후보 영역
5. 근거 라벨과 음성 답변

질문:

> 제가 지금 있는 곳은 대동여지도에서 어디였나요?

답변:

> 천안군 권역 후보로 읽힐 가능성이 높습니다. 정확한 한 점이 아니라 옛 행정권역 기준의 추정입니다.

### 4. 근거형 도슨트 화면

목표는 "AI가 그냥 말하는 것"이 아니라는 점을 보여주는 것입니다.

| 카드 | 내용 |
| --- | --- |
| 판본 근거 | 국립중앙박물관 대동여지도 신수19997, 해당 첩 영역, 도로, 읍치 기호 |
| 공공데이터 근거 | e뮤지엄, 국가유산청, 문화공공데이터 |
| 추정 | 옛 행정권역 후보, 신뢰도 0.78, 권역 단위 추정 |

강조 문장:

> 근거 없으면 단정하지 않음

### 5. GYEOL Engine 구조도

복잡한 기술 구조보다 5개 모듈을 간단히 보여줍니다.

- VIEW: 실제 판본 보기
- VOICE: 음성 RAG 도슨트
- ASK: 자연어 질문
- GEO: 옛지도-현대좌표 정합
- DATA: 공공 문화데이터 정규화

하단 문장:

> 대동여지도에서 검증한 뒤, 고지도·고문헌형 유산으로 확장

### 6. 기존 서비스 대비 차별표

| 기존 전시·구글 | 결(GYEOL) |
| --- | --- |
| 고해상으로 본다 | 기호를 짚어 해독한다 |
| 녹음 해설을 듣는다 | 근거형 AI와 대화한다 |
| 지도 자체를 본다 | 내 위치를 옛지명으로 번역한다 |
| 콘텐츠마다 새로 제작 | 공통 엔진 + 유형별 어댑터 |

### 7. 기대효과

| 대상 | 효과 |
| --- | --- |
| 어린이 | 쉬운 말로 묻고 이해 |
| 외국인 | 다국어 도슨트 |
| 고령층 | 음성 중심 해설 |
| 지역주민 | 내 동네의 옛 이야기 |

핵심 문장:

> 읽을 수 있는 사람만 이해하던 유산을, 묻기만 하면 되는 유산으로 바꾼다.

### 8. MVP 로드맵

발표에서는 2주, 1개월, 3개월 타임라인으로 보여줍니다.

- 2주: 1첩, 기호 5개, 질문 20개, 천안/서울/부산 3지역
- 1개월: 22첩 확장, 다국어 3종, 신뢰도 라벨
- 3개월: 박물관 키오스크, 교사용 모드 PoC

## 발표에서 피할 것

- 깨진 AI 생성 텍스트
- 출처 없는 역사 단정
- 너무 많은 데이터 표
- "모든 국가유산 즉시 확장" 같은 과장
- 대동여지도를 AI가 새로 그린 것처럼 보이는 화면
- API 키나 개인정보
- 너무 긴 영상

## 최종 한 장

발표자료 중 가장 중요한 한 장은 다음 구성을 기준으로 합니다.

질문:

> 제가 지금 있는 곳은 대동여지도에서 어디였나요?

입력:

> 현재 위치: 천안역

처리:

> 현재 좌표 → 옛지명 후보 → 대동여지도 영역 → 신뢰도 산출

답변:

> 천안군 권역 후보로 읽힐 가능성이 높습니다. 단, 정확한 한 점이 아니라 권역 단위 추정입니다.

근거:

- 판본 근거: 국립중앙박물관 대동여지도 신수19997
- 공공데이터 근거: e뮤지엄, 국가유산청, 문화공공데이터
- 추정: 신뢰도 0.78, 권역 단위
