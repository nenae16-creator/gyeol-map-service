# MVP 로드맵

결(GYEOL)의 MVP는 대동여지도 전체를 한 번에 구현하는 것이 아니라, 검증 가능한 범위를 좁혀 "현재 위치 → 옛지명 후보 → 판본/공공데이터 근거 → 신뢰도 라벨" 흐름을 먼저 증명하는 데 집중합니다.

## MVP 원칙

- 실제 판본 이미지를 기반으로 한다.
- AI가 새 지도를 생성하거나 복원하는 것처럼 보이지 않게 한다.
- 단일 정답보다 후보와 신뢰도를 제시한다.
- 판본 근거, 공공데이터 근거, 추정을 분리해 보여준다.
- 발표와 시연에서는 개인정보와 API 키를 노출하지 않는다.

## 2주 목표

범위:

- 대동여지도 1첩
- 핵심 기호 5개
- 대표 질문 20개
- 천안, 서울, 부산 3개 지역

기능:

- 현재 위치 입력
- 현대 행정구역·POI 변환
- 옛지명 후보 3개 표시
- 대동여지도 후보 영역 표시
- 근거 라벨 표시
- 권역 단위 추정 답변

산출물:

- 발표용 핵심 데모 화면
- 대표 질문/답변 세트
- 데이터 출처 목록
- 신뢰도 라벨 기준 초안

## 1개월 목표

범위:

- 22첩 확장
- 다국어 3종
- 신뢰도 라벨 고도화

기능:

- 지도표 기호 해설 확장
- 다국어 도슨트 답변
- 후보별 근거 비교
- 낮은 신뢰도일 때 단정하지 않는 답변 정책

산출물:

- 확장 데이터셋 구조
- 다국어 답변 샘플
- 신뢰도 산출 기준
- 사용자 흐름 프로토타입

## 3개월 목표

범위:

- 박물관 키오스크 PoC
- 교사용 모드 PoC
- 고지도·고문헌형 유산 확장 가능성 검토

기능:

- 음성 중심 질의응답
- 어린이/외국인/고령층 대상 답변 톤 분기
- 수업용 질문 세트
- 전시 환경용 화면 모드

산출물:

- 키오스크 PoC
- 교사용 모드 PoC
- 확장 어댑터 설계 초안
- 운영 및 데이터 갱신 계획

## 우선순위

1. 천안역 핵심 데모의 공식 근거·판면 확대 흐름을 유지하고 전문가 좌표 검수를 진행한다.
2. 근거 라벨과 추정 라벨을 화면에서 명확히 분리한다.
3. 실제 판본 이미지를 기준으로 후보 영역을 보여준다.
4. 신뢰도가 낮을 때의 답변 정책을 먼저 정한다.
5. 다국어, 키오스크, 교사용 모드는 핵심 흐름 검증 후 확장한다.

현재 프로토타입은 천안역의 한국철도공사 대표점, 국가철도공단의 1905년 개업 기록, 천안시 1416년 천안군 연혁, 국립중앙박물관 신수19997 공식 이미지 파일 96의 천안 표기를 연결했습니다. 좌표 동일시를 피하기 위해 결과와 확대 화면 모두 `권역 대응 추정` 및 지오리퍼런싱 한계를 표시합니다.

## 핵심 성공 기준

심사위원이 다음 문장을 이해하면 MVP의 목적은 달성됩니다.

> 결은 대동여지도를 크게 보여주는 서비스가 아니라, 현재 위치를 옛지명 후보와 연결하고 판본/공공데이터 근거로 설명하는 해독 엔진이다.
