# 결(GYEOL)

현재 내 위치를 1861년 대동여지도의 기호, 길, 옛지명으로 번역하는 공공 문화데이터 기반 AI 문화유산 해독 엔진입니다.

## 프로젝트 요약

결(GYEOL)은 대동여지도를 단순히 크게 보여주는 서비스가 아닙니다. 사용자가 "제가 지금 있는 곳은 대동여지도에서 어디였나요?"라고 물으면 현재 위치를 옛 지명 후보와 연결하고, 실제 판본 이미지와 공공 문화데이터 근거를 바탕으로 답변하는 서비스입니다.

핵심 전환은 다음과 같습니다.

- 보는 지도에서 묻고 답하는 지도로
- 감상하는 지도에서 해독하는 지도로
- 옛 지도에서 내 위치와 연결되는 지도로

## 기획상 핵심 데모

질문:

> 제가 지금 있는 곳은 대동여지도에서 어디였나요?

입력 예시:

- 현재 위치: 천안역

처리 흐름:

1. 현재 위치 확인
2. 현대 행정구역과 주변 POI 변환
3. 옛지명 후보 3개 추출
4. 대동여지도 해당 영역과 교차검증
5. 신뢰도 산출
6. 판본 근거, 공공데이터 근거, 추정 라벨과 함께 음성 답변

답변 예시:

> 지금의 천안역은 대동여지도 기준으로 천안군 권역 후보로 읽힐 가능성이 높습니다. 다만 정확한 한 점이 아니라 옛 행정권역 기준의 추정입니다.

중요한 원칙은 단정하지 않는 것입니다. 결은 "정확히 여기다"라고 말하지 않고, 근거와 신뢰도를 함께 제시하며 권역 후보로 추정합니다.

## 현재 프로토타입

기본 화면은 영상형 지도 연출보다 서비스의 핵심 질문과 근거형 답변을 먼저 보여주는 4단계 흐름으로 구성되어 있습니다.

1. 현재 위치 또는 장소 질문
2. 현대 위치와 옛지명 권역 후보 대조
3. 판본·공공데이터·추정을 구분한 결과
4. 사용자가 선택한 뒤 관련 지도 전개 체험

현재 시범 데이터는 서울시청·광화문·경복궁 인근의 `한양·한성부 권역 후보` 한 사례만 제공합니다. 지원하지 않는 위치는 가장 가까운 옛지명을 임의로 반환하지 않고 시범 범위 밖이라고 안내합니다.

- 실제 검색 폼과 현재 위치 권한 요청
- 현대 장소 → 옛지명 후보 → 판본 확인 → 근거형 답변의 순차 상태
- 숫자 신뢰도 대신 `판본에서 확인`, `공공데이터 확인`, `권역 대응 추정` 라벨
- 국립중앙박물관 대동여지도 신수19997 소장품 출처 연결
- 결과 화면에서 사용자가 선택할 때만 기존 접이 지도·먹길·김정호 체험 실행
- 체험 먹길은 `한 획 = 체험 눈금 1칸`으로 전개하고 10칸마다 이정표를 표시합니다. 체험 30칸은 실제 거리 단위가 아닙니다. 현대 거리는 [공주시청 공식 위치 페이지](https://www.gongju.go.kr/kr/sub05_04_01.do)에 내장된 공주시청 기준점에서 선택 목적지까지의 대권 직선거리와 [카카오모빌리티 자동차 길찾기](https://developers.kakaomobility.com/guide/navi-api/directions)의 추천 경로를 구분해 표시합니다. 길찾기 키가 없거나 요청에 실패하면 직선거리는 유지하고 자동차 경로만 `현재 확인할 수 없음`으로 안내합니다. 고지도 노정 거리는 별도로 `현재 확인되지 않음`으로 표시합니다. 원본 지도는 도로에 10리마다 점을 찍었다는 [국립중앙박물관 소장품 설명](https://www.museum.go.kr/MUSEUM/contents/M0502000000.do?relicId=4502&schM=view&searchId=search)을 참고했습니다.
- 기존 화면은 소스·자산 백업과 `?legacy=1` 접근 경로로 보존

실행:

```bash
npm install
npm run dev -- --port 5187
```

접속:

```text
http://127.0.0.1:5187
```

기존 지도 체험 화면:

```text
http://127.0.0.1:5187/?legacy=1
```

검증:

```bash
npm run typecheck
npm run verify:road-distance
npm run build
npm run qa
npm run qa:production
npm run smoke:road-distance -- --url=https://배포주소
```

Vercel 보호 미리보기는 `VERCEL_AUTOMATION_BYPASS_SECRET`을 서버 환경에서 제공하거나, 인증된 CLI의 `npx --yes vercel@latest curl /api/road-distance?... --deployment https://미리보기주소`로 확인합니다. 우회 토큰을 명령행·문서·클라이언트 코드에 직접 넣지 않습니다.

`npm run qa`는 로컬 서버를 대상으로 데스크톱·모바일의 질문, 해독, 근거 결과, 체험 진입·복귀, 미지원 위치와 위치 권한 거부 상태를 검사하고 `tmp/qa/`에 결과를 저장합니다. `npm run qa:production`은 동일한 검사를 운영 주소에서 실행하고 `tmp/qa-production/`에 저장합니다.

모든 화면의 이미지 로드 상태, 콘솔·페이지 오류, 실패한 요청, HTTP 오류와 브라우저 요청의 민감 키 이름 노출을 검사하며 하나라도 실패하면 종료 코드 1을 반환합니다. 다른 배포 주소를 검사할 때는 `QA_BASE_URL`, `QA_ALLOW_REMOTE=1`과 `tmp/` 내부의 `QA_OUTPUT_DIR`을 사용할 수 있습니다.

## 구현 구조

- `src/components/GyeolServiceExperience.tsx`: 질문, 현재 위치, 해독, 근거 결과, 체험 연결
- `src/components/GyeolServiceExperience.module.css`: 사진형 작업대 위 서비스 레이아웃과 상태별 전환

- `api/road-distance.js`: 카카오 REST 키를 보호하는 자동차 추천 경로 서버 함수
- `src/data/roadDistanceClient.ts`: 동일 출처 도로거리 요청과 성공·실패 상태 정규화
- `src/components/InteractiveDaedongMapIntro.tsx`: 결과 이후에 실행되는 기존 지도 전개 체험
- `public/assets/gyeol-service-desk-plate.png`: 검색·진행 문구를 비운 서비스용 사진 배경판
- `backups/2026-07-10-current-map-experience/`: 서비스 개편 전 화면의 소스·설정·사용 자산 백업

결은 공공 문화데이터를 답변 근거로 사용합니다.

- 국립중앙박물관 e뮤지엄 소장품 데이터
  - 대동여지도 신수19997 실제 판본 이미지
  - 소장품 메타데이터
  - 판본 근거 라벨
- 국가유산청 고지도·지도표 해설
  - 지도표 14개 항목, 22종 기호 해설
  - 도로, 읍치, 역참, 봉수, 능침 등 해독
- 문화공공데이터광장 Open API
  - 박물관 및 문화자원 메타데이터 보강
  - 공공데이터 근거 라벨
- 자체 전국 POI 데이터
  - 현재 위치 주변 맥락 보조
  - 옛지명 후보 매칭 보조

## GYEOL Engine

결은 재사용 가능한 엔진 구조를 목표로 합니다.

- VIEW: 실제 대동여지도 판본 이미지를 보여주고 필요한 부분을 레이어로 점진 공개합니다.
- VOICE: 사용자의 음성 질문을 받아 STT, RAG, TTS 구조로 답변합니다.
- ASK: 사용자가 자연어로 자유롭게 질문합니다.
- GEO: 옛지도와 현대 좌표를 연결하고 후보 3개와 신뢰도를 제공합니다.
- DATA: 국립중앙박물관, 국가유산청, 문화공공데이터 등 공공 문화데이터를 정규화합니다.

## 발표 메시지

심사위원이 30초 안에 이해해야 하는 문장은 다음입니다.

> 결은 대동여지도를 크게 보여주는 서비스가 아니라, 지금 내 위치를 1861년 대동여지도의 기호, 길, 지명으로 번역하는 공공 문화데이터 해독 엔진입니다.

발표에서 반복할 키워드는 다음 5개입니다.

- 현재 위치
- 옛지명 후보
- 대동여지도 실제 판본
- 판본 근거 / 공공데이터 근거 / 추정
- 묻기만 하면 읽히는 문화유산

## 표현 원칙

피해야 할 표현:

- AI가 대동여지도를 복원한다
- 천안역은 정확히 대동여지도에서 천안군이다
- 모든 국가유산에 바로 적용 가능하다
- 대동여지도를 새로 그린다

사용해야 할 표현:

- 실제 판본 이미지를 기반으로 해독한다
- 권역 단위 후보로 추정한다
- 고지도·고문헌형 유산부터 확장한다
- 판본 근거, 공공데이터 근거, 추정을 분리해서 보여준다
