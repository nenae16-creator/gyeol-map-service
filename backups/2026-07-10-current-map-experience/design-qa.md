# Design QA

## Findings

- [P1] 인앱 브라우저 목록이 비어 있어 실제 렌더링 기반 최종 시각 QA는 차단됨.
  - 범위: 한성부 실제 클릭, 중앙 지도 이동·먹색 융화 프레임, 8면 seam, 보정된 먹길 접점, 상세 지도 후순위 노출, 김정호 보행, 건너뛰기·다시 보기, 포커스, 콘솔.
  - 영향: 자산·좌표·정적 합성·타임라인·빌드는 검증했지만 브라우저의 실제 프레임 합성과 상호작용은 확정할 수 없음.
  - 후속: 인앱 브라우저 연결 후 1487×1058에서 첫 화면과 최종 화면을 각각 기준 이미지와 캡처해 비교하고 전체 흐름을 재생해야 함.

## Full-View Comparison Evidence

- 첫 화면은 사진형 작업대의 가로 한지 위에 레퍼런스 윤곽을 유지한 투명 중앙 지도를 약 `(510,127)–(1022,894)`로 배치함.
- 개선 전/후 비교에서 어두운 목재 위의 큰 좌우 공백을 좌측 벼루·붓·두루마리, 우측 고서·먹물 항아리와 넓은 한지 면으로 채웠으며, 지도에는 배경의 우상단 광량과 따뜻한 종이 톤을 반영함.
- 헤더·검색창·하단 진행 단계와 바닥은 고정되고, 첫 화면 사진형 한지판과 중앙 지도는 서로 독립된 오브젝트로 분리됨.
- 최종 화면은 좌측 지도 배경 `(43,111)–(815,928)`, 접이 지도 `(0,362)–(872,880)`, 우측 상세 지도 `(650,121)–(1487,931)`를 기준으로 배치함. 접이 지도 우측 먹선과 상세 지도 서쪽 먹선은 약 `(872,602)`에서 만나도록 자산과 보행 좌표를 함께 보정함.
- 먹길 25·50·75·100% 정적 비교에서 다른 지도 요소는 고정되고, 길이 제거된 기본판 위에 원본 먹선이 좌측부터 연속적으로 복원되는 것을 확인함.
- 최종 비교본의 김정호는 기준 이미지에 없지만 사용자가 요청한 보행 기능을 보여주기 위한 의도적 차이임. 도착 도장은 런타임 도착 상태에만 표시됨.
- 두 비교 파일은 정적 합성이며 실제 브라우저 캡처가 아니므로 최종 결과는 blocked로 유지함.

## Focused Region Comparison Evidence

- 중앙 지도는 `512 × 767` 톤 보정 컷아웃을 약 `(510,127)–(1022,894)`에 `object-fit: contain`으로 배치함.
- 중앙 한성부 hotspot 중심: 약 `(689,465)` = `(46.34%,43.95%)`, hit area 최대 `72 × 72px`.
- 클릭 후 첫 화면 사진형 한지판은 `0.45–1.45s`에 고정된 채 부드럽게 사라지고, 중앙 지도만 `0.00–1.55s`에 `xPercent -75.5`, `yPercent -7.3`, `scale 0.9`, `rotateY -10deg`, `rotateZ 4.5deg`로 비율을 유지하며 기존 좌측 종착점으로 이동함.
- 일반 지도와 동일한 먹색 multiply 레이어를 교차해 `0.42–1.55s`에 채도·명암·흐림을 배경 한지와 맞추고, 좌측 배경판에는 22px 페더 마스크를 적용해 `0.58–1.53s` 교차 전환의 절단 경계와 별도 그림자를 제거함.
- 접이 지도는 가시 영역 `(228,153,1187,641)`과 실제 주름 x좌표 `[228,430,605,713,847,995,1142,1268,1415]`를 기준으로 나눈 8개 면이 `1.64–3.72s`에 전개되고 `4.30s`까지 안착·교차 전환됨.
- 각 주름은 전체 `1607×979` 자산을 8회 렌더링하지 않고 원본과 픽셀 차이가 없는 개별 패널 텍스처로 분리했으며, 패널 blur를 제거해 3D 전개 부하를 줄임.
- 원본 v4에서 분리한 먹선 오버레이와 보행 경로 seam을 정규화 `y≈0.463`으로 맞춰 상세 지도 먹선과 약 `(872,602)`에서 이어지도록 보정함.
- ImageGen 복원 지면을 좁은 먹길 코리더에만 합성한 기본판 위로, 같은 마스크로 분리한 원본 먹선·인접 픽셀을 `3.90–6.25s`에 34px 선단으로 드러냄. 인공 직선·대각 연장 없이 최종 상태는 원본 v4/v2 경로를 복원함.
- 단일 접이 지도 교차 전환과 패널 소멸이 `4.30s`에 끝난 뒤, 한성부 상세 지도 tween이 `4.50s`에 시작해 `5.85s`에 끝남. 확대 원점은 경로 접점에 대응하는 `25.3% 59.4%`임.
- 상세 지도와 먹선 노출 완료 뒤 김정호가 `6.42–18.92s`에 63개 실측 먹길 중심점을 따라 이동하고, 한성부 내부에서는 보이는 시가지 길을 따라 도착점으로 이동함. 프레임마다 `left/top` 대신 transform 좌표만 갱신함.

## Required Fidelity Surfaces

- Typography: 헤더·지도·진행 단계의 글자는 래스터 자산을 그대로 사용함. 상호작용 라벨만 프로젝트의 명조 계열 스택을 사용함.
- Spacing: 중앙·좌측·접이·상세 지도 위치를 1487×1058 실측 좌표에서 백분율로 변환함.
- Colors: 기준 이미지의 목재·한지·먹·인주색을 원본 및 편집 자산에서 유지함.
- Image quality: 레퍼런스 중앙 지도, 먹길 제거 `base-v5`·`base-v3`, 원본 먹선 오버레이를 투명 PNG로 분리했으며 중앙 지도 윤곽과 두 지도의 연결 좌표를 코드의 가시 영역과 함께 보정함.
- Accessibility: 실제 button, `aria-live`, 최소 44px hit target, 사진형 오프닝 한지판과 페더 마스크를 포함한 17개 자산 준비 전 선택 비활성화, 로드 오류 안내, 건너뛰기, 다시 보기, reduced-motion, 시작·도착·재시작 시 포커스 인계를 코드에서 확인함.

## Comparison History

1. 좌측 지도가 베이크된 첫 화면을 빈 고정 책상판과 투명 중앙 지도 종이로 분리함.
2. 첫 화면 중앙 지도 배치를 기존 오프닝 시안과 같은 좌표로 맞춤.
3. 클릭 시 책상은 고정하고 중앙 지도만 왼쪽으로 이동한 뒤 원래 최종 시안의 좌측 지도 배경과 교차 전환하도록 변경함.
4. 접이 지도 8개 면의 실제 주름 전개는 유지함.
5. 한성부 상세 지도 tween을 초기 구간에서 제거하고 접이 전개·교차 전환이 완전히 끝난 뒤로 이동함.
6. 상세 지도 노출이 완료된 뒤에만 김정호 보행을 시작하도록 전체 타임라인을 순연함.
7. `positioning`과 `revealing` 상태를 추가하고 skip·ARIA·reduced-motion·replay 복원을 새 순서에 맞춤.
8. 최종 배경판을 좌측 종이 polygon으로 제한해 우측 나무결·헤더·하단 띠는 빈 고정판에서 계속 유지되도록 보정함.
9. 자산 로드 실패 상태를 추가해 영구 로딩 대신 새로고침 안내를 제공함.
10. 중앙 지도를 `daedongyeojido-reference-cutout.png`로 교체하고 한성부 hotspot을 새 윤곽에 맞춰 이동함.
11. 일반 지도와 multiply 먹색 복제본을 교차해 좌측 배경 한지로 융화되는 전환을 추가함.
12. 접이 지도를 `gyeol-folded-route-cutout-v4.png`로 교체하고 마지막 먹선·보행 seam을 상세 지도 접점 `(872,602)`에 맞춤.
13. 8면 전개를 `1.64–4.30s`, 상세 지도 확대를 `4.50–5.85s`, 보행을 `6.42–18.92s`로 조정함.
14. 좌측 배경판의 polygon 절단과 별도 drop-shadow를 22px 페더 마스크로 교체함.
15. 실제 먹선 픽셀 오버레이를 추가해 `3.90–6.25s`에 길이 번지도록 하고, 63개 실측 좌표로 김정호 동선을 맞춤.
16. 접이 텍스처를 실제 주름별 8개 무손실 crop으로 분리하고 패널 blur를 제거했으며 보행을 transform 갱신으로 변경함.
17. 첫 화면 전용 사진형 한지판을 추가하고 지도를 배경 광원에 맞춘 톤 보정본으로 교체해 좌우 공백과 스티커처럼 떠 보이는 인상을 줄임.
